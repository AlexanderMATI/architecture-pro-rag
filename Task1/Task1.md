# Задание 1. Исследование моделей и инфраструктуры для RAG-бота

## 1. Сравнение LLM-моделей 

### 1.1. Локальные модели (Hugging Face)

Варианты: 

#### Llama 2/3, Mistral, Falcon, MPT

Преимущества:
- полный контроль над данными;
- нет ограничений по токенам;
- предсказуемые затраты после развертывания;

Недостатки:
- высокиеинфраструктурные затраты;
- необходимость GPU для приемлемой скорости.

Метрики:
- скорость: 20-50 токенов/сек (с GPU);
- качество: 70-85% от GPT-4 (зависит от модели);

#### OpenAI (GPT-4, GPT-3.5-turbo):

Преимущества:
- лучшее качество ответов на рынке;
- быстрое развертывание

Недостатки:
- данные передаются третьей стороне (риск для SOC 2);
- непредсказуемые затраты при масштабировании;

Метрики:
- скорость: 40-100 токенов/сек;
- качество: эталонное (GPT-4);


#### YandexGPT:

Преимущества:
- соответствие российским требованиям (если актуально);
- конкурентная цена;
- поддержка русского языка.

Недостатки:
- ограниченная документация на английском;
- меньшее сообщество и экосистема;

## 3. Сравнение векторных баз данных

### 3.1 FAISS (Facebook AI Similarity Search) — выбрана для проекта

**Преимущества:**
- экстремально быстрый поиск (миллисекунды);
- отличная оптимизация памяти (индексы IVF, PQ);
- работает локально (безопасность данных);
- бесплатно и open-source;
- проверено в production (Meta, OpenAI).

**Недостатки:**
- только in-memory или файловая система;
- нет встроенной персистентности;
- отсутствует multi-tenancy;
- требуется самостоятельная реализация CRUD;
- нет нативной фильтрации по метаданным.

**Метрики (для 1024 dim, как в проекте):**
- скорость поиска: <10ms для 1M векторов;
- скорость индексации: ~5000 векторов/сек;
- использование памяти: ~2.1GB для 100K векторов;
- сложность внедрения: средняя (нужна обертка);
- стоимость: €0.

**Почему FAISS выбран:** высокая производительность, локальное развертывание, полный контроль над данными.

---

### 3.2 ChromaDB

**Преимущества:**
- удобный API (out-of-the-box persistence);
- встроенная фильтрация по метаданным;
- нативная интеграция с LangChain;
- поддержка multi-tenancy;
- встроенное управление коллекциями;
- может работать как сервер (client-server mode).

**Недостатки:**
- медленнее FAISS при больших масштабах;
- больше overhead по памяти;
- относительно молодой проект.

**Метрики (для 1024 dim):**
- скорость поиска: ~50-100ms для 1M векторов;
- скорость индексации: ~2000 векторов/сек;
- использование памяти: ~3.5GB для 100K векторов;
- сложность внедрения: низкая (ready-to-use);
- стоимость: €0 (self-hosted).

## 6. Итоговые рекомендации

### Рекомендуемое решение: Вариант 3 (Гибридный)

**Обоснование:**

1. **Соответствие текущим задачам:**
   - Быстрое получение результата (дни, а не месяцы)
   - Низкий порог входа (бесплатные инструменты + минимальные затраты на API)

2. **Баланс безопасности:**
   - Векторная индексация локально (данные не покидают периметр)
   - Только запросы и релевантные фрагменты передаются в YandexGPT

3. **Технические преимущества:**
   - FAISS: максимальная скорость поиска, бесплатно
   - Multilingual-E5-large: отличное качество, поддержка русского языка
   - YandexGPT: низкая стоимость, хорошее качество

4. **Финансовая эффективность:**
   - Затраты на API: ~€20/мес (при 100 запросах/день)
   - Инфраструктура: ~€200/мес (лёгкая VM)
   - Экономия времени сотрудников: €1,000-2,000/мес
   - **ROI: 3-6 месяцев**
